Week 10

More Testing

Acceptance Testing
Acceptance testing is done by the user/client. Does the user accept the product? Is the user satisfied with the product?

  • Tested by passing driver cases or a few sample cases.

Correctness Testing
Correctness testing is done by the software designers. It makes sure that your algorithms behave appropriately to all allowable inputs.

  • Tested by identifying all cases and creating tests for all of them.
  • Your implemented data structure must represent the correct structure at its current state.

Testing Strategies

  • Before testing anything, you want to know what kind of input your program will be receiving. What ranges of values? Will you get invalid input?
  • Think of a few cases that will properly represent a “general” case that your program will receive. Implement those tests.
  • Think of “edge” cases, which don’t happen as often as your “general” cases but your program has to properly handle those cases as well.

Memory Model Review

Key facts to remember:

  • Every time a variable is declared, a “box” is allocated to store information. That allocated location in memory is then stored.
  • Variables are local within the function they are declared in, so each function has its own space in memory.
